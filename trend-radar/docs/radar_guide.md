# Nura TikTok Viral Screening Radar — Руководство

## Принцип работы

Скрипт собирает TikTok-видео из трёх групп источников (конкуренты, хештеги, ключевые слова + ротационный пул), оценивает их вирусный потенциал и генерирует отчёт с AI-анализом топ-10.

### Этапы

1. **Сбор** — Playwright открывает страницы TikTok, перехватывает API-ответы (полная статистика) или вытягивает данные из DOM/SSR-разметки
2. **Обогащение** — для видео без статистики заходит на страницу `/video/ID` и вытягивает SSR JSON с просмотрами, лайками, автором
3. **Скóринг** — считает метрики вирусности (Engagement Rate, Viral Score, Final Score)
4. **AI-анализ** — топ-10 отправляются в DeepSeek (генерация 3 сценариев для TikTok + Instagram-карусели)
5. **Отчёт** — Markdown + XLSX (метрики) + XLSX (сценарии в формате продакшн-таблицы) + `.txt` с полным AI-разбором

## Настройка

### Конфигурационные файлы (`config/`)

| Файл | Назначение |
|------|-----------|
| `competitors.txt` | Username конкурентов (без @). Заполняется вручную. Одна строка = один аккаунт. |
| `hashtags.txt` | Core-хештеги — всегда в обходе. Одна строка = один хештег (без #). |
| `keywords.txt` | Core-ключевые слова — всегда в обходе. Одна строка = один поисковый запрос. |
| `rotational.txt` | Ротационный пул. Одно слово → хештег, несколько слов → ключевое слово. Меняется раз в день. |

Дополнительно в корне проекта:
- `Ротационные ключевики.txt` — план ротации на 7 дней (не подключается к коду, используется как источник для ручного обновления `rotational.txt`).

### .env

| Параметр | Значение | По умолчанию | Описание |
|----------|----------|-------------|----------|
| `MIN_VIEWS` | число | 10000 | Мин. просмотров для попадания в топ |
| `MAX_RESULTS_PER_SOURCE` | число | 20 | Макс. видео с одного источника |
| `HEADLESS` | true/false | true | Показывать браузер (false — видно окно) |
| `CHROME_DEBUG_PORT` | порт | — | Подключение к живому Chrome |
| `LOG_LEVEL` | INFO/DEBUG | INFO | Уровень логов |

### HEADLESS=false (рекомендовано)

Скрипт запускает видимый браузер. Удобно чтобы видеть, что происходит. Скрипт сам загружает куки, сам их обновляет при следующем запуске.

### Автообновление кук (если протухли)

При старте скрипт проверяет валидность кук. Если они протухли:
1. Открывает страницу входа в TikTok
2. Ждёт пока ты войдёшь (проверяет URL каждые 2 сек, макс 5 мин)
3. Сохраняет свежие куки автоматически
4. Продолжает сбор

Никаких ручных операций — просто войди в аккаунт в открывшемся окне.

## Факторы, влияющие на результат

### 🟢 Решаемо

| Фактор | Решение |
|--------|---------|
| playList в топе | `is_playlist` флаг + фильтр |
| networkidle таймаут | fallback `load` → `domcontentloaded` |
| AI на пустых данных | фильтр по `views is None` |
| Хештеги без контента | пропускаются, кеш блокировки удалён |
| Скриншоты забивают диск | Только DEBUG-режим + автоочистка 7 дней |
| DOM-видео без метрик | enrichment через `/video/ID` |
| AI-анализ не виден | отдельные `.txt`-файлы + XLSX-таблица |
| AI-запрос подвис | обработка ошибки, fallback |
| Куки протухли | авто-логин при старте, сохранение свежих кук |
| Футажи не совпадают по цвету | `VideoOverlay.color_grading` — цветокоррекция каждого футажа отдельно |

### 🔴 Внешние (не решаются кодом)

| Фактор | Влияние |
|--------|---------|
| TikTok изменил SSR JSON | Править `_find_items`, `_find_items_detail` |
| TikTok изменил API | Править `on_response` фильтр + `_parse_item` |
| TikTok изменил CSS-классы | Править `_extract_from_dom` |
| Российский IP | VPN для доступа к /tag/ |
| Медленный интернет | goto таймауты — увеличить таймауты в .env |
| Время дня | Результаты отличаются утром/вечером |
| Тренды меняются | Другие видео — это нормально |

## Итоговые файлы

После запуска в `data/`:

| Файл | Содержание |
|------|-----------|
| `reports/report_YYYY-MM-DD_HH-MM.md` | Полный отчёт (Markdown) |
| `reports/report_YYYY-MM-DD_HH-MM.xlsx` | Таблица с метриками (Excel) |
| `ai_analysis/01_videoid_caption.txt` | Полный AI-анализ по каждому из топ-10 |
| `scenarios/scenarios_YYYY-MM-DD_HH-MM.xlsx` | Сценарии в формате продакшн-таблицы (TikTok + Instagram) |
| `videos.db` | SQLite-база всех собранных видео |
| `tiktok_cookies.json` | Сохранённые куки (автоматически) |
| `debug/*.png` | Скриншоты (только при LOG_LEVEL=DEBUG) |

### Формат сценариев (scenarios_*.xlsx)

| Колонка | Назначение |
|---------|-----------|
| A. ID видео | Внутренний идентификатор |
| B. Формат | Тип видео (говорящая голова / сплит-экран / анимация) |
| C. Цель | Маркетинговая цель |
| D. Локация / Фон | Описание фона и света |
| E. Крючок (Текст на экране) | Первое, что видит зритель |
| F. Полный скрипт озвучки (Речь Нуры) | Текст для TTS/HeyGen (15-30 сек) |
| G. Стоковые вставки | Описание стоковых видео и триггеров |
| H. Динамика камеры и эффекты | Зумы, переходы, динамика |
| I. Стиль и позиция субтитров | Шрифт, цвет, позиция |
| J. Директива для Карусели (Instagram) | 5-8 слайдов для Instagram |
| K. Переход | dissolve / slide / fade / zoom |
| L. Темп (BPM) | Темп музыки (100-160) |
| M. Тональность | тёплый / мистический / энергичный / минималистичный / драматичный |

Колонки B–I → для **TikTok** (сценарий видео).
Колонка J → для **Instagram** (карусель на ту же тему).

## Видеосборщик (nura_app)

Проект `nura_app` содержит `services/video_assembler.py` — движок сборки видео по JSON-сценарию (FFmpeg).

### Маппинг таблицы → JSON сборщика

| Колонка таблицы | Поле JSON |
|----------------|-----------|
| E (Крючок) | `scenes[].overlays[].text` |
| F (Скрипт озвучки) | `scenes[].nura_text` |
| G (Стоковые вставки) | `scenes[].overlays[].video.src` |
| I (Субтитры) | `subtitles` config |
| K (Переход) | `scenes[].transition.type` |
| Цветокоррекция футажа | `scenes[].overlays[].video.color_grading` |

### Цветокоррекция футажей

Каждый футаж (`VideoOverlay`) поддерживает `color_grading`:

```json
{
  "src": "videos/media/stock.mp4",
  "color_grading": {
    "enabled": true,
    "midtones_red": 0.08,
    "highlights_blue": 0.05,
    "glow": true,
    "glow_radius": 12,
    "glow_opacity": 0.15
  }
}
```

Применяется ДО наложения футажа на сцену — футаж подгоняется под цветовую гамму Nura, основное видео не затрагивается.

## Если что-то пошло не так

1. **`Error at ...: Timeout`** — проверь интернет
2. **`Blocked at ... login`** — куки протухли. В открывшемся окне просто войди в TikTok, скрипт сам подхватит
3. **`0 videos (DOM)` у всех** — запусти с `HEADLESS=false`, посмотри что в окне
4. **Топ из плейлистов** — удали `data/videos.db` и запусти заново
5. **AI не анализирует** — проверь `DEEPSEEK_API_KEY` в `.env`, баланс на аккаунте DeepSeek
6. **XLSX не создаётся** — `pip install openpyxl`
7. **Имя файла с ?** — исправлено, санитизация символов Windows

## Мониторинг

Раз в месяц запускать с `LOG_LEVEL=DEBUG` и проверять:
- `Got N videos (API)` — API-перехват работает
- `Stats enriched: N/N` — enrichment даёт метрики
- Нет новых WARNING/ERROR паттернов

Если API перестал работать для поиска — обновить `_find_items` под новую структуру SSR JSON.
