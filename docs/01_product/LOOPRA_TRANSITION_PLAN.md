# LOOPRA Transition Plan

## Version

v1.0

## Status

Brand and Architecture Migration Plan

------------------------------------------------------------------------

# 1. Purpose

This document defines the transition from the historical project name
Content Plant to the new product brand LOOPRA.

The goal is not only renaming.

The goal is aligning:

-   product identity;
-   architecture;
-   documentation;
-   development workflow;
-   future platform vision.

------------------------------------------------------------------------

# 2. Previous Identity

Historical name:

Content Plant

Original concept:

A system for automated content production.

Initial focus:

-   trend discovery;
-   content generation;
-   video assembly;
-   production workflows.

The project evolved beyond this initial scope.

------------------------------------------------------------------------

# 3. New Identity

Current brand:

LOOPRA

Category:

Autonomous Marketing Operating System

Core idea:

A system that continuously improves brand growth through intelligent
marketing loops.

------------------------------------------------------------------------

# 4. What Remains Unchanged

The following principles remain valid:

-   project-agnostic foundation;
-   separation of core and project logic;
-   filesystem-first MVP approach;
-   controlled development phases;
-   deterministic tools;
-   validation before expansion.

Existing Foundation MVP remains the technical base.

------------------------------------------------------------------------

# 5. What Changes

## Terminology

Old:

Content Plant

New:

LOOPRA

Old:

Content production platform

New:

Autonomous Marketing Operating System

Old:

Content workflow

New:

Growth Loop

------------------------------------------------------------------------

# 6. Documentation Migration

## Active Documents

Become LOOPRA source of truth:

-   LOOPRA_ARCHITECTURE.md
-   LOOPRA_BRAND_POSITIONING.md
-   LOOPRA_ROADMAP.md
-   AGENTS.md
-   STATE.md
-   README.md

## Archive

Historical documents move to:

docs/archive/content-plant-era/

Purpose:

Preserve historical context without confusing active development.

------------------------------------------------------------------------

# 7. Repository Migration Rules

During transition:

-   no unnecessary code renaming;
-   no architecture rewrite;
-   no feature expansion;
-   no SaaS implementation.

The migration is documentation and identity alignment first.

------------------------------------------------------------------------

# 8. Agent Instructions

AI coding agents must:

-   use LOOPRA as current project name;
-   treat Content Plant as historical terminology;
-   follow LOOPRA architecture documents;
-   avoid implementing future vision without explicit tasks.

------------------------------------------------------------------------

LOOPRA
